# 数据格式

数据格式（format）描述多维 Tensor 轴的业务语义，表示数据的物理排布格式。

## 常见数据格式

| 格式 | 维度 | 说明 |
|------|------|------|
| ACL_FORMAT_ND | N维 | 低维度优先连续排布（通用格式，大部分算子支持） |
| ACL_FORMAT_NCHW | 4D | N(Batch), C(Channel), H(Height), W(Width) |
| ACL_FORMAT_NHWC | 4D | N, H, W, C 排布 |
| ACL_FORMAT_HWCN | 4D | H, W, C, N 排布 |
| ACL_FORMAT_NDHWC | 5D | N, D(Depth), H, W, C |
| ACL_FORMAT_NCDHW | 5D | N, C, D, H, W |
| ACL_FORMAT_NC | 2D | N, C |
| ACL_FORMAT_NCL | 3D | N, C, L(Length) |

## 维度与格式约束

- **5D Tensor**：NCDHW / NDHWC / ND (如 API 支持)
- **4D Tensor**：NCHW / NHWC / HWCN / ND
- **3D Tensor**：NCL / ND
- **2D Tensor**：NC / ND
- **其他维度**：仅 ND

## NPU 私有格式

不常用，绝大多数 aclnn API 不支持：
- `ACL_FORMAT_NC1HWC0` — 5D 分形格式
- `ACL_FORMAT_FRACTAL_Z` — 4D 分形格式
- `ACL_FORMAT_FRACTAL_NZ` — ND 分形格式（权重常用）
- `ACL_FORMAT_NC1HWC0_C04` — C04 分形
- `ACL_FORMAT_NDC1HWC0` — ND 5D 分形
- `ACL_FORMAT_FRACTAL_Z_3D` — 3D 分形

> 若个别 API 声明支持上述私有格式，以该 API 实际描述为准

## 维度缩写

| 缩写 | 含义 |
|------|------|
| N | Batch（批量大小） |
| H | Height（特征图高度） |
| W | Width（特征图宽度） |
| C | Channels（特征图通道） |
| D | Depth（特征图深度） |
| L | Length（特征图长度） |
