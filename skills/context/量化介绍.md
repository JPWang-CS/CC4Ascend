# 量化介绍

## 概述

CANN 算子量化：对 Matmul 等 Cube 类算子的输入 Tensor 从高 bit 到低 bit 转换，同时生成 scale 量化参数。低 bit 计算完成后通过 scale 还原，保证精度近似等价，提升计算效率。

## 量化类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 静态量化 | 预先确定量化参数，性能更好 | 推理场景 weight 量化 |
| 动态量化 | 在线计算量化参数，适应数据变化、精度更高 | 推理/训练场景 activation 量化 |

## 量化模式（量化粒度）

- **m, n, k** = Tensor 的各轴大小
- **左矩阵** = 激活 activation（shape: [m, k]）
- **右矩阵** = 权重 weight（shape: [k, n]）
- **k** = reduce 轴

### pertensor（T 量化）

整个 Tensor 共用 1 个 scale。scale shape: `(1,)`

### perchannel（C 量化）

量化对象：右矩阵。每个 channel 独立 scale。scale shape: `(n,)`

### pertoken（K 量化）

量化对象：左矩阵。每个 token 独立 scale。scale shape: `(m,)`

### pergroup（G 量化）

在 reduce 轴 k 上分组，每组独立 scale。group size = gs。
- 左矩阵 scale shape: `(m, k/gs)`
- 右矩阵 scale shape: `(k/gs, n)`

### perblock（B 量化）

在**所有轴**上分块，每块独立 scale。block size = bs。
- 左矩阵 scale shape: `(m/bs, k/bs)`
- 右矩阵 scale shape: `(k/bs, n/bs)`

## 常见组合量化

| 名称 | 模式 | 说明 |
|------|------|------|
| 全量化 T-C | pertensor-perchannel | 左矩阵 T + 右矩阵 C |
| 全量化 K-C | pertoken-perchannel | 常用推理模式 |
| 全量化 G-B | pergroup-perblock | |
| 全量化 T-CG | pertensor-perchannel-pergroup | |
| 全量化 B-B | perblock-perblock | |
| 伪量化（C量化） | perchannel | 仅对 weight 量化 |
| Mx量化（G-G） | pergroup-pergroup | FLOAT8_E8M0, group size=32 的特例 |

## 量化应用场景

- **推理**：全量化 / 伪量化 → 降低访存、加速计算
- **A5 (950)**：新增 MXFP8/MXFP4 量化支持，更高吞吐（757T@FP8 vs 426T@FP16）
- **动态量化**：在线生成 scale，性能略差但精度更高
