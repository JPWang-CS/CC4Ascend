# A2A3 (Ascend 910B/910C) 硬件规格

## 概览

| 规格项 | 参数 |
|--------|------|
| AI Core 总数 | 24 |
| HBM 容量 | 64 GB |
| L2 容量 | 192 MB |

## Cube 单元

| 规格项 | 参数 |
|--------|------|
| AIC 版本 | AIC-C-220 |
| CCEC_AIC 版本 | dav-c220-cube |
| Cube 核数量 | 24 |
| L0A 大小 | 64 KB |
| L0B 大小 | 64 KB |
| L0C 大小 | 128 KB |
| Bias 大小 | 1 KB |
| 是否支持 Fixpipe | 是 |
| 是否支持 Sparsity | 是 |

## Vector 单元

| 规格项 | 参数 |
|--------|------|
| CCEC_AIV 版本 | dav-c220-vec |
| Vector 核数量 | 48 |
| UB 大小 | 192 KB |

## 与 A5 关键差异

| 维度 | A2A3 (910B) | A5 (950) |
|------|-------------|----------|
| AI Core 数量 | 24 | 36 (+50%) |
| HBM 容量 | 64 GB | 128 GB (+100%) |
| L2 容量 | 192 MB | 128 MB (-33%) |
| L0C 大小 | 128 KB | 256 KB (+100%) |
| Bias 大小 | 1 KB | 4 KB (+300%) |
| UB 大小 | 192 KB | 248 KB (+29%) |
| Vector 核数 | 48 | 72 (+50%) |
| Fixpipe | 支持 | 不支持 |
| Sparsity | 支持 | 不支持 |

## 设计要点

- **Fixpipe 可用**：A2A3 支持硬件 Fixpipe，可直接利用 L0C→L0A/L0B 的直连通路做数据搬运
- **Sparsity 可用**：支持结构化稀疏的硬件加速，可设计 sparse MatMul 路径
- **同地址冲突**：A2A3 硬件不支持同地址请求并行处理，分核策略需要错位规避
- **L0C 较小 (128KB)**：tile 粒度受限于 L0C 容量，K 方向需要更多循环轮次
- **UB 较小 (192KB)**：双缓冲/三缓冲的 tile 粒度门限更低
- **L2 较大 (192MB)**：相比 A5 有更大的 L2 缓存，可以容纳更多的数据复用
- **Cube:Vector = 1:2**：每个 AI Core 含 1 个 Cube 核 + 2 个 Vector 核
