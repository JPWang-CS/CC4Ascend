## 与 A2A3 关键差异

| 维度 | A2A3 (910B) | A5 (950) |
|------|-------------|----------|
| AI Core 数量 | 24 | 36 (+50%) |
| HBM 容量 | 64 GB | 128 GB (+100%) |
| L2 容量 | 192 MB | 128 MB (-33%) |
| L1 大小 | — | 512 KB |
| L0C 大小 | 128 KB | 256 KB (+100%) |
| Bias 大小 | 1 KB | 4 KB (+300%) |
| UB 大小 | 192 KB | 248 KB (+29%) |
| Vector 核数 | 48 | 72 (+50%) |
| Fixpipe | 支持 | 不支持 |
| Sparsity | 支持 | 不支持 |

## 设计要点

- **L0C 翻倍 (128→256KB)**：允许更大的 tile 粒度，减少 K 方向循环次数
- **UB 扩大 (192→248KB)**：更多片上缓冲空间，双缓冲/三缓冲阈值上调
- **Fixpipe 不可用**：A5 不支持硬件 fixpipe，需要使用软件流水替代
- **Sparsity 不可用**：结构化稀疏由硬件加速不可用，需走 Dense 计算路径
- **L2 缩小 (192→128MB)**：需更注意 L2 数据复用策略，避免频繁 DDR 回刷
- **HBM 翻倍 (64→128GB)**：可容纳更大模型，但带宽不变仍需关注访存效率
